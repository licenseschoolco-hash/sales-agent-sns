# SNS営業OS 運用マニュアル

## 1. 営業OSの思想と目的

### 思想：AIが下書きし、人間が魂を込めて送る
本システムは「自動送信ツール」ではありません。AIは情報の整理と文章の下書き（ドラフト）を行い、**最終的な送信判断、内容の修正、送信作業は必ず人間が行う**「Human-in-the-loop」設計です。SNSという人間同士の信頼関係が重要なチャネルにおいて、誠実で質の高いコミュニケーションを実現することを目的としています。

### 目的
- SNSリード獲得から商談成約までを一気通貫で管理する。
- 営業活動を数値化し、ボトルネック（詰まっている箇所）を特定して改善する。
- 「次回対応予定日」の管理により、リードの取りこぼし（追撃漏れ）をゼロにする。

## 2. 1日の標準運用フロー

### 朝：状況把握と優先順位付け（15分）
1.  **ダッシュボード確認** (`/social-leads/dashboard`)
    - 改善アクションを確認し、「PDF送付率」や「商談化率」が低下していないかチェック。
2.  **パイプライン確認** (`/targets`)
    - 「⚡ 本日対応」と「⚠️ 期限超過」の企業を抽出。
    - 本日アプローチすべきターゲットのリストを頭に入れる。

### 日中：攻めの営業活動（随時）
1.  **新規リード登録** (`/social-leads`)
    - SNSで見つけた見込み客を随時登録。
2.  **DM送信サイクル** (`/social-leads/[id]`)
    - `DM生成` → `生成文を人間が確認・修正` → `SNS上で手動送信` → `TouchLogを記録`。
3.  **反応への対応**
    - 返信があったら即座に `TargetCompany` へ昇格。
    - 興味関心に合わせて `AI診断レポート` を作成し、レポートURLを添えて手動DMを送付。

### 夕方：記録と翌日の仕込み（30分）
1.  **商談結果の更新** (`/targets/[id]`)
    - 本日の商談結果、メモ、金額を保存し、`Appointment` 履歴を残す。
2.  **次回対応日の設定**
    - 全ての交渉中案件に対し、必ず「次回対応予定日」を入力する。
3.  **KPIの振り返り**
    - 本日の送信数やアポ獲得数が目標に届いているかダッシュボードで確認。

## 3. 画面ごとの使い方

### /social-leads (リード一覧)
- **用途**: 新規リードの発見と登録。
- **操作**: 「+ 新規リード登録」からアカウント情報を入力。

### /social-leads/[id] (リード詳細)
- **用途**: 初回接触の実行。
- **操作**: 
    - 「DM生成」でアプローチ文を作成。
    - **[重要]** 生成された文章をコピーし、SNSのDM画面に貼り付ける。
    - 送信後、必ず「TouchLogを追加」で送信内容を記録する。

### /social-leads/dashboard (KPI分析)
- **用途**: 営業戦略の修正。
- **操作**: 期間（7d/30d）を切り替え、各フェーズの転換率を確認。改善コメントに従ってDM文面などを調整する。

### /targets (パイプライン一覧)
- **用途**: 追撃管理。
- **操作**: サマリーカードの「本日対応」をクリックし、漏れがないか確認する。

### /targets/[id] (企業詳細・商談管理)
- **用途**: CRM（顧客管理）と成約記録。
- **操作**: 「⚡ クイック営業管理」パネルで、商談メモと金額を記録する。

### /targets/[id]/reports/ai (AI診断作成)
- **用途**: 価値提供（フック）の作成。
- **操作**: 診断タイプを選択して「診断レポートを生成」。生成されたレポート内容が企業の現状と乖離していないか確認する。

## 4. 営業判断の基準

| アクション | 判断基準 |
| :--- | :--- |
| **DMを送る** | ターゲット属性（業種・規模）が合致し、かつ直近でSNS活動がある場合。 |
| **TargetCompanyへ昇格** | 返信があった、またはプロフィールから明らかに「話を聞く価値がある」と判断した場合。 |
| **PDF送付に進む** | 初回DMで「診断に興味がある」という反応があった、または興味を持ってもらうための「手土産」が必要な場合。 |
| **Zoom誘導（アポ打診）** | レポート送付後に「詳しく知りたい」「自社の場合はどうなるか」等の質問が出た場合。 |
| **成約(won)にする** | 発注書を受領、または契約合意が取れ、金額が確定した時点。 |
| **NGにする** | 営業拒否の意思表示があった、または競合他社であると判明した場合。 |

## 5. 安全ルール・禁止事項（※厳守）

1.  **自動送信の禁止**: SNSアカウントの凍結リスクを避けるため、DM送信作業は必ず「手動」で行うこと。
2.  **AI文章の未確認送信禁止**: AIは誤った情報を生成することがあります。必ず人間が読み直し、不自然な敬語や誤字を修正してから送信してください。
3.  **業界特有の配慮（重要）**: 医療・介護・歯科業界向けの場合、**「必ず成功する」「売上が倍増する」といった断定表現や誇大表現は避け**、「〜の傾向があります」「〜という事例があります」といった柔軟な表現に修正してください。
4.  **TouchLogの徹底**: 「何を送ったか」が記録されていないと、チーム内での重複連絡や矛盾した提案の原因になります。
5.  **SNS API非連携の維持**: システムから直接SNSを操作することはありません。ブラウザの別タブでSNSを開いて運用してください。

## 6. KPIの見方と対策

- **DM送信率が低い**: リード登録数に対してアプローチが追いついていない。リストアップの基準を緩めるか、作業時間を確保する。
- **PDF送付率が低い**: 初回DMの「診断オファー」が魅力的に映っていない。AI生成時の「商品メリット」の設定を見直す。
- **成約率が低い**: 商談の質、またはリードのターゲット適合性に問題がある。TargetCompanyへの昇格基準を厳しくする。
- **Overdue（期限超過）が多い**: 追撃（フォローアップ）が営業の最優先事項であることを再認識し、朝一番に処理する。

## 7. 運用チェックリスト

### 朝チェック
- [ ] /targets で「期限超過」が0件になっているか？
- [ ] 「本日対応」のタスクは何件あるか把握したか？

### DM送信前チェック
- [ ] 宛名（会社名・担当者名）は間違っていないか？
- [ ] AIが作った「嘘」や不自然な表現はないか？
- [ ] 医療・介護系の場合、断定表現を使っていないか？

### PDF送付前チェック
- [ ] 診断レポートの「改善点」が具体的で納得感があるか？
- [ ] レポートURLが正しく開けるか？

### 商談後チェック
- [ ] `/targets/[id]` に商談メモを残したか？
- [ ] **次回の対応予定日を必ず入力したか？**

### 夕方チェック
- [ ] 本日発生した全ての「反応」をシステムに記録したか？
- [ ] 翌日のアポ予定を確認したか？

---

## 8. よくある運用ミスと対策

- **ミス**: TouchLogを付け忘れて、同じ人に2回同じDMを送った。
  - **対策**: 送信ボタンを押した直後に、必ずTouchLog画面を開く癖をつける。
- **ミス**: 次回対応日を入れ忘れて、リードが放置された。
  - **対策**: 詳細画面の保存ボタンを押す前に「次回対応日が入っているか」を指差し確認する。

## 9. まだやらないこと（現在のフェーズ外）

- SNSアカウントの自動操作（フォロー・いいね・DM送信）。
- Googleカレンダーとの自動同期。
- Zoom URLの自動発行。
- 広告費の管理や売上予測グラフ。

## 10. 次フェーズの候補

- **SNS投稿生成機能**: 営業DMだけでなく、自身のSNSアカウントを育てるための投稿案作成。
- **簡易カンバン表示**: パイプラインを視覚的にドラッグ＆ドロップで管理。
- **チーム管理機能**: 複数名の営業担当者によるアサイン管理。
