# ワークフロー・ルール

- **計画の提示**: 次フェーズの計画提示は、確認で止まらず「計画のみ」をまず出力する。
- **実装のタイミング**: 明示的な指示があるまでコード変更は行わない。
- **外出中作業モード**: TASK に記載された低リスクな作業のみを自律的に進める。
- **エラー対応**: エラー修正の試行は最大3回まで。
- **失敗時の報告**: 3回失敗した場合は、原因、試したこと、次の候補案を報告して作業を停止する。
- **コミット前の停止**: コミット操作を行う前に必ずユーザーの確認を得る。
- **状態確認**: 作業前後で必ず `git status` を確認し、意図しない変更がないかチェックする。
