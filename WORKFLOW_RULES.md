# ワークフロー・ルール

- **計画の提示**: 次フェーズの計画提示は、確認で止まらず「計画のみ」をまず出力する。
- **実装のタイミング**: 明示的な指示があるまでコード変更は行わない。
- **外出中作業モード**: TASK に記載された低リスクな作業のみを自律的に進める。
- **エラー対応**: エラー修正の試行は最大3回まで。
- **失敗時の報告**: 3回失敗した場合は、原因、試したこと、次の候補案を報告して作業を停止する。
- **コミット前の停止**: コミット操作を行う前に必ずユーザーの確認を得る。
- **状態確認**: 作業前後で必ず `git status` を確認し、意図しない変更がないかチェックする。

## プロジェクト定義と責任範囲：自社営業エージェント

本プロジェクトは、弊社の自社商品を販売するための「営業エージェント（自社営業部）基盤」である。
外部販売用の「営業OSプロダクト」そのものではない。

すべての開発判断は、「商談・アポの創出効率が上がるか」を最優先基準とする。

### 営業エージェントの責務

本プロジェクトで実装してよい範囲は以下とする。

- SNSアカウント発掘
- SNS投稿生成
- 見込みアカウント探索
- SNS DM文生成
- 無料診断誘導
- PDF診断レポート生成
- 商材別課題仮説の提示
- 返信・アポ管理
- LINE/Zoom誘導
- 電話履歴の管理
- LeadCandidate から TargetCompany への昇格管理
- 商談・成約につながる営業導線の改善

### 明示的な責任範囲外

以下は本プロジェクトでは実装しない。

- 公式問い合わせフォームへの自動送信機能
- 公式問い合わせフォーム送信履歴管理
- 既存の問い合わせフォーム営業OSとの完全統合
- AI秘書そのものの実務実行機能
  - 仕訳
  - 請求書発行
  - 書類整理の実働
  - 税理士連携
  - タスク管理の実働画面
- 営業OSプロダクトとしての外部顧客アカウント管理
- 課金管理
- マルチテナントSaaS管理
- 商品戦略エージェントが担当する市場分析、原価計算、競合比較管理

### SNS営業モジュールの責務

本プロジェクト内のSNS営業モジュールは、SNS内での関係構築からアポ獲得までの導線に特化する。

担当範囲：
- SNSアカウント発掘
- 投稿生成
- 投稿結果管理
- 見込みアカウント探索
- フォロワースコアリング
- いいね、コメント、フォロー、DMの接触履歴管理
- SNS DM文生成
- 無料診断誘導
- PDF送付管理
- LINE/Zoom誘導
- 返信・アポ管理

公式問い合わせフォーム営業は、既存の問い合わせフォーム営業OSが担当する。
本プロジェクトでは、公式問い合わせフォーム送信を実装しない。

### 開発時の意思決定ルール

新機能を追加する前に、必ず以下を確認する。

1. その機能は、自社商品の商談・アポ獲得に直接つながるか
2. 商品そのものの機能開発になっていないか
3. 既存の問い合わせフォーム営業OSと責務が重複していないか
4. AI秘書プロダクトや営業OSプロダクトの本体機能を作ろうとしていないか
5. SNS営業モジュールの責務を超えていないか

上記に反する場合は、本プロジェクトには実装せず、商品戦略エージェントまたは該当プロダクト側の課題として切り分ける。
