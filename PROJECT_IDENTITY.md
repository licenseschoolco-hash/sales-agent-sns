# PROJECT_IDENTITY

このプロジェクトは 営業エージェントSNS／自社営業部エージェント です。

対象ディレクトリ：
/Users/takeshiyamashita/営業エージェントSNS

このプロジェクトの目的：
自社商品を販売するための営業活動を支援すること。
SNS上で見つけた見込み客を、DM・診断レポート・商談管理・KPI分析を通じて、アポ・成約につなげるための自社営業基盤です。

このプロジェクトで扱うもの：
- SNSリード登録
- SNSリード管理
- SocialTouchLogによる接触履歴管理
- AIによるDM下書き生成
- 初回接触DM
- 無料診断誘導DM
- PDF送付DM
- Zoom誘導DM
- TargetCompanyへの昇格
- SNSプロフィール情報のInformationSource化
- AI診断レポート作成導線
- PDFレポート送付導線
- TargetCompanyの営業ステータス管理
- Appointmentによる商談履歴管理
- アポ・成約・失注・金額・次回対応日の記録
- SNS営業KPIダッシュボード
- TargetCompany一覧の営業パイプライン表示
- 自社営業運用マニュアル

このプロジェクトで扱う自社商品：
- AI秘書
- 営業OS
- AI電話
- AIアニメ動画
- 歯科AI電話
- 採用アニメ動画
- 高齢者シェアハウス
- 身元保証
- その他、自社が営業対象として販売する商品

このプロジェクトで扱わないもの：
- AI秘書プロダクト本体の開発
- 経理AIエージェント本体の開発
- 領収書・請求書アップロード機能
- 銀行履歴・カード明細照合機能
- 税理士提出前整理機能
- AI電話プロダクト本体の開発
- AI動画制作プロダクト本体の開発
- 外販SaaSとしての営業OSプロダクト本体
- 外部顧客向けの課金・権限管理
- マルチテナント管理
- SNS APIによる自動DM送信
- SNS APIによる自動投稿
- 自動いいね
- 自動フォロー
- Google Calendar連携
- Zoom URL自動発行
- 請求書発行
- 決済管理

責務の定義：
このプロジェクトは「自社商品を売るための営業エージェント」です。
商品そのものを開発する場所ではありません。
商品戦略を最終決定する場所でもありません。
外部企業に提供する営業OSプロダクト本体でもありません。

役割分担：
- 営業エージェントSNS：
  自社商品を売るための営業活動、リード管理、DM下書き、診断導線、商談管理、KPI確認を担当する。

- AI秘書プロダクト：
  中小企業の経理・事務・バックオフィス整理を支援する外販商品であり、本プロジェクト内では実装しない。

- 営業OSプロダクト：
  外部企業に提供する営業支援SaaSであり、本プロジェクトとは別管理とする。

- 商品戦略エージェント：
  価格、訴求、ターゲット、市場選定、競合比較を設計する役割であり、本プロジェクトに商品本体機能を混ぜない。

最重要ルール：
すべての開発判断は、
「自社商品のアポ・商談・成約につながるか」
を基準に判断する。

禁止ルール：
- 営業に直接関係しない商品本体機能を追加しない
- AI秘書の経理機能を混ぜない
- AI電話の通話実行機能を混ぜない
- AI動画制作の納品管理機能を混ぜない
- 外販SaaS用の課金・権限管理を混ぜない
- 自動DM送信を実装しない
- 自動投稿を実装しない
- SNS API連携を勝手に追加しない
- DB変更やmigrationは明示指示なしに行わない
- prisma/dev.db / prisma/dev.db-journal はコミットしない
- git add . は使用しない

安全設計：
AIは営業文面の下書き、診断の補助、KPIの可視化を行う。
最終送信、投稿、商談判断、成約判断は必ず人間が行う。
医療・介護・歯科領域では断定表現、誇大表現、効果保証表現を避ける。
AI生成文は必ず人間が確認・修正してから使用する。

このプロジェクトの完成形：
SNSで見込み客を発見し、
SNSリードとして登録し、
AIでDM下書きを作成し、
人間が手動で送信し、
反応があればTargetCompanyへ昇格し、
AI診断レポートを作成し、
PDF_SEND DMで手動送付し、
Zoom誘導し、
TargetCompany詳細で商談・成約・失注を記録し、
KPIダッシュボードで成果を確認し、
次の営業改善につなげる。
